# Security Policy

## Supported Versions

Only the latest release of the JIT Access Portal is actively supported with security updates. We recommend always running the latest tagged version.

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of this project seriously. If you believe you have found a security vulnerability, please do **not** report it via a public issue. Instead, report it privately to our security team.

Please send your vulnerability reports to: **security-disclosures@example.com** (replace with your organization's actual security contact email).

Please include the following details in your report:
* The name and version of the component affected (frontend, backend, database driver, etc.).
* A detailed description of the vulnerability and its potential impact.
* Step-by-step instructions (and ideally a proof of concept script/request payload) to reproduce the issue.
* Any potential mitigations you have identified.

Once received, the security team will:
1. Acknowledge receipt of the report within 48 hours.
2. Investigate the issue and coordinate a fix.
3. Prepare a security patch and release a new version.
4. Provide credit to the reporter (unless anonymity is requested) in the release notes.

Thank you for helping keep this project secure!
